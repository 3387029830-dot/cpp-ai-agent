# AI 编程智能体 —— 需求规格说明书

> 项目代号：cpp-ai-agent  
> 版本：v1.1  
> 日期：2026-07-06

## 1. 项目背景

AI Coding Agent 是一种能够理解用户开发任务，并通过工具调用完成文件操作、命令执行、代码分析等工作的智能体系统。

本项目面向课程实践和答辩演示，使用 C++17 实现一个运行在终端中的 AI 编程智能体。项目重点不追求完整复刻成熟产品，而是展示软件工程项目从需求、设计、编码、测试到交付的完整过程。

## 2. 项目目标

本项目需要实现一个可编译、可运行、可演示的 C++ 智能体程序，具备以下能力：

- 接收用户输入的自然语言编程任务。
- 调用 OpenAI 兼容格式的大语言模型 API。
- 支持多轮对话上下文。
- 支持模型发起工具调用。
- 支持文件读取、文件写入、局部编辑和命令执行。
- 对高风险操作进行权限确认。
- 记录执行日志，并支持历史回放。
- 通过终端界面展示对话、状态、工具调用和结果。

## 3. 用户与使用场景

| 用户 | 使用场景 |
|------|----------|
| 学生开发者 | 使用智能体辅助理解代码、读写文件、运行命令 |
| 答辩评委 | 观察项目架构、运行效果、工程文档和测试结果 |
| 后续维护者 | 根据模块划分继续扩展工具、模型适配和界面 |

典型流程：

```text
用户输入任务
→ 模型理解任务
→ 模型判断是否需要调用工具
→ 系统检查权限
→ 工具执行
→ 结果回填给模型
→ 模型生成最终回答
→ TUI 展示全过程
```

## 4. 术语定义

| 术语 | 说明 |
|------|------|
| Agent | 智能体，负责任务编排、模型调用和工具调用 |
| LLM | 大语言模型，例如 OpenAI 兼容 API 提供的模型 |
| Tool | 工具，智能体可调用的原子能力 |
| Tool Call | 模型返回的工具调用请求 |
| Session | 会话，一次连续对话的上下文 |
| TUI | 终端文本界面 |
| Skill | 更高层的能力封装，可由提示词和工具组合形成 |
| MCP | Model Context Protocol，本项目只做最小概念演示 |

## 5. 功能需求

| 编号 | 功能 | 优先级 | 验收方式 |
|------|------|--------|----------|
| F1 | 用户输入与上下文管理 | P0 | 连续输入两轮问题，上下文保持一致 |
| F2 | LLM API 调用 | P0 | 配置 API Key 后能获得真实模型回复 |
| F3 | Agent 主循环 | P0 | 模型能发起工具调用并获得工具结果 |
| F4 | 文件读取工具 | P0 | 能读取工作目录内文件并返回内容 |
| F5 | 文件写入与编辑工具 | P0 | 能写入新文件，能按查找替换编辑文件 |
| F6 | Shell 命令工具 | P0 | 能执行允许的命令并返回输出和退出码 |
| F7 | 权限控制 | P0 | 高风险操作执行前必须确认 |
| F8 | 执行日志 | P0 | 用户输入、模型回复、工具调用写入 JSON |
| F9 | TUI 展示 | P0 | 能展示对话区、工具区、状态区和输入区 |
| F10 | 单元测试 | P1 | 核心工具和路径校验有测试用例 |
| F11 | Web 搜索工具 | P1 | 可接 API；无 API 时提供占位说明 |
| F12 | Skill 管理 | P1 | 能从配置文件加载简单 Skill |
| F13 | MCP 演示 | P2 | 提供最小 mock 客户端或接口占位 |

## 6. 权限分级

| 风险等级 | 操作类型 | 处理方式 |
|----------|----------|----------|
| Safe | 读取文件、查看配置 | 直接执行并记录 |
| Write | 写入文件、编辑文件 | 默认确认后执行 |
| Dangerous | 删除文件、执行命令 | 强制确认，可拒绝 |

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 平台 | 优先支持 Windows 11，尽量保持跨平台设计 |
| 构建 | 使用 CMake 管理项目，支持命令行构建 |
| 可维护性 | 模块职责清晰，新增工具不修改 Agent 主循环 |
| 可测试性 | 工具、路径校验、日志模块可单元测试 |
| 安全性 | API Key 不硬编码，日志中敏感信息脱敏 |
| 可演示性 | 每个里程碑都有可运行产物 |
| 文档性 | README、需求、设计、计划与实际代码保持一致 |

## 8. 约束与假设

- 用户已具备可用的大模型 API Key。
- 首次拉取依赖需要网络。
- 当前课程场景以演示和学习为主，不处理企业级权限隔离。
- MCP、Web 搜索和 Skill 属于扩展功能，优先保证 P0 主线完整。

## 9. 验收标准

- CMake 配置和构建成功。
- 程序可以运行，并展示终端界面。
- 可以真实调用大模型 API。
- 可以由模型触发文件读取工具。
- 写入、命令执行等高风险操作有确认机制。
- 执行过程能保存到 JSON 日志。
- 至少包含路径安全、文件工具、工具注册表相关测试。
- README 能让他人在新环境中复现构建和运行。

一、粉红色 —— 时标对象（Moment-Interval）
核心问题：系统需要追踪/记录/审计什么？

粉红色是四色建模的起点，代表时刻或时间段上发生的事情。

1. Session（会话） — MI 顶层容器
文件: Session.h
属性: id_（会话ID）、messages_（消息列表）
含义: 用户从启动程序到输入 /exit 的整个对话时段。这是最顶层的 Moment-Interval，所有其他 MI 都嵌套其中。
1 : N 关系: 一个 Session 包含多条 Message
2. Message（消息） — MI 核心载体
文件: Message.h
属性: role（角色）、content（内容）、toolCalls（工具调用列表）、toolCallId（关联的工具调用ID）、timestamp（时间戳）
含义: 对话流中的每一条消息——用户发言、系统提示、模型回复、工具返回。这是系统的关键业务记录。
1 : N 关系: 一条 Assistant Message 可能包含多个 ToolCall
3. ToolCall（工具调用） — MI 嵌套细节
文件: Message.h:16-20
属性: id（唯一标识）、name（工具名）、arguments（参数 JSON）
含义: 大模型请求调用某个工具的瞬间。需要被追踪（日志记录）、被审计（权限确认）、被回放（历史重放）。
生命周期: 发起 → 权限确认 → 执行 → 返回结果
4. PermissionRequest（权限请求） — MI 安全关卡
文件: PermissionManager.h:16-21
属性: toolName、risk、arguments、preview
含义: 工具执行前的权限确认时刻。可被批准（approve）或拒绝，决定了 ToolCall 是否能继续执行。
5. AgentEvent（运行时事件） — MI 流式通知
文件: AgentLoop.h:17-28
属性: type（AssistantMessage / AssistantChunk / AssistantDone / ToolCall / ToolResult / Warning）、title、detail
含义: Agent 主循环执行过程中发出的细粒度事件，驱动 Console 的流式展示。AssistantChunk 携带 SSE 文本增量，AssistantDone 表示流式结束。
二、黄色 —— 角色对象（Role）
核心问题：参与方以什么身份参与时标对象？

黄色总是依附于某个粉红色的 Moment-Interval 存在。

1. Role（消息角色枚举）
文件: Message.h:9-14
取值: System（系统指令）、User（用户）、Assistant（模型）、Tool（工具返回）
依附关系: 每个 Message（粉红）都有一个 Role（黄），决定了消息在对话中的身份。
扮演者:
System → 程序自身
User → 终端操作者
Assistant → LlmClient（外部 LLM 服务）
Tool → 各种 ITool 实现
2. RiskLevel（风险等级角色）
文件: ITool.h:9-13
取值: Safe（安全，如读文件）、Write（写入，如编辑文件）、Dangerous（危险，如执行 Shell 命令）
依附关系: 每个 ToolCall（粉红）执行时，其对应的 ITool（蓝/绿）被赋予一个风险等级（黄），决定 PermissionManager 如何处理。
3. PermissionMode（权限模式角色）
文件: PermissionManager.h:10-14
取值: AskEachTime（每次都问）、ReadOnly（只读模式）、TrustSession（信任本会话）
依附关系: 整个 Session（粉红）期间的权限策略角色，决定 PermissionRequest 的处理方式。
三、蓝色 —— 描述对象（Description）
核心问题：用什么目录/规范来分类和描述事物？

蓝色是"是什么类型"的目录信息，独立于具体实例存在。

1. ITool（工具接口描述） — 核心蓝本
文件: ITool.h
描述属性: name()、description()、parametersSchema()、risk()
含义: 定义了"什么是工具"的模板——任何工具必须提供名称、描述、参数 schema 和风险等级。这是工具目录的 Catalog Entry。
2. ToolRegistry（工具注册表/目录） — 目录集合
文件: ToolRegistry.h
含义: 所有可用工具的目录。LLM 通过 toolsSpec() 获取这个目录来决定调用哪个工具。
3. McpTool（MCP 工具描述）
文件: McpClient.h:10-14
描述属性: name、description、inputSchema
含义: 从外部 MCP 服务器发现的工具的目录条目。
4. McpServerInfo（MCP 服务器描述）
文件: McpClient.h:16-22
描述属性: protocolVersion、name、version、instructions、tools[]
含义: 外部 MCP 服务器的目录信息，包含其提供的工具清单。
5. AppConfig / LlmConfig（配置描述）
文件: AppConfig.h:10-16, LlmClient.h:12-17
描述属性: LLM 连接信息（baseUrl、apiKey、model、proxyUrl）、maxIterations、permissionMode、workspaceRoot、historyDir
含义: 描述"系统应该如何运行"的规范。
6. DiagnosticItem（诊断项描述）
文件: Diagnostics.h:10-14
描述属性: status、name、detail
含义: 描述某个诊断检查的结果。
7. HistoryFile（历史文件描述）
文件: HistoryReader.h:9-13
描述属性: path、size、modifiedTime
含义: 描述一个历史会话日志文件的元数据。
四、绿色 —— 参与方/地点/物品（Party/Place/Thing）
核心问题：谁/什么实际参与了这些时刻？

绿色是可以承担黄色角色、被蓝色描述的具体实体。

外部参与方（Party）
类	文件	说明
LlmClient	LlmClient.h	外部 LLM 服务，在消息中扮演 Assistant 角色。通过 HTTP 与 OpenAI 兼容 API 通信
物品/执行者（Thing）
类	文件	说明
AgentLoop	AgentLoop.h	核心编排器，协调 LLM 调用、工具执行、事件发射的循环
PermissionManager	PermissionManager.h	权限决策实体，根据 RiskLevel 和 PermissionMode 决定是否放行工具调用
ContextManager	ContextManager.h	上下文窗口管理器，控制发送给 LLM 的消息数量
JsonLogger	JsonLogger.h	将 Session 中的所有 MI 事件持久化为 JSONL
ReadFileTool	FileTools.h	读取工作区内文件
WriteFileTool	FileTools.h	写入/覆盖工作区内文件
EditFileTool	FileTools.h	精确字符串替换编辑文件
ShellTool	ShellTool.h	执行 Shell 命令
McpToolAdapter	McpToolAdapter.h	将外部 MCP 工具适配为 ITool 接口
StdioMcpClient	McpClient.h	通过 stdio 与外部 MCP 服务器通信的客户端
地点/界面（Place）
类	文件	说明
Console	Console.h	终端 UI，呈现 Banner、消息、工具调用、权限提示、警告/错误

## 2. 根目录文件

根目录放的是“项目级配置文件”。这些文件通常不放进子目录，因为对应工具默认会在项目根目录查找它们。

| 文件 | 作用 |
|------|------|
| `CMakeLists.txt` | CMake 的项目构建说明 |
| `CMakePresets.json` | CMake 的预设配置，统一构建命令 |
| `vcpkg.json` | vcpkg 的依赖清单 |
| `README.md` | 项目入口说明 |
| `AGENTS.md` | 项目级 Agent 规范文件，启动时注入 system prompt |
| `.gitignore` | Git 忽略规则 |
| `.env.example` | 环境变量样例 |
| `.clang-format` | 代码格式化规则 |

## 3. 子目录

| 目录 | 作用 |
|------|------|
| `config/` | 放程序运行配置，比如模型、权限、Skill 配置 |
| `docs/` | 放项目文档 |
| `src/` | 放 C++ 源代码 |
| `tests/` | 放单元测试代码 |
| `build/` | 放 CMake 生成文件和 exe，可删除重建，不提交 Git |

## 4. 为什么不把所有配置都放进 config/

`config/` 只放“程序运行时自己读取的配置”。

例如：

- `settings.json`
- `skills.json`

而下面这些文件是“开发工具读取的配置”，通常必须或推荐放在根目录：

- `CMakeLists.txt`
- `CMakePresets.json`
- `vcpkg.json`
- `.gitignore`
- `.clang-format`

所以它们不是乱放，而是按工具约定放在根目录。

## 5. 后续结构演进

当前 `src/` 已按功能拆分：

```text
src/
├── main.cpp
├── agent/
├── config/
├── core/
├── diagnostics/
├── history/
├── llm/
├── mcp/
├── security/
├── skills/
├── storage/
├── tools/
└── ui/
```

这些目录对应 M0-M8 的主要能力：模型对话、工具系统、Agent 主循环、安全日志、Skill、Web 搜索、MCP 和 ANSI 流式控制台。

当前主要模块：

| 路径 | 作用 |
|------|------|
| `src/agent/` | Agent 主循环，负责模型工具调用、工具结果回填和多轮控制 |
| `src/config/` | 配置加载和 provider 模板 |
| `src/core/` | 消息、会话和上下文窗口 |
| `src/diagnostics/` | `/config`、`/doctor` 等诊断能力 |
| `src/history/` | 历史日志读取和回放 |
| `src/llm/` | OpenAI-compatible API 客户端和 `tool_calls` 解析 |
| `src/mcp/` | 最小 MCP stdio 客户端和 MCP 工具适配 |
| `src/security/` | 权限确认、风险分级、危险命令拦截和 diff preview |
| `src/skills/` | Skill 配置加载、提示词注入和工具白名单 |
| `src/storage/` | JSONL 会话日志 |
| `src/tools/` | 本地工具，包括文件、目录、命令和 Web 搜索 |
| `src/ui/` | ANSI 流式控制台和 `/ui` 概览 |
| `src/tools/ITool.h` | 工具统一接口、风险等级和工具结果结构 |
| `src/tools/ToolRegistry.*` | 工具注册、查找和 OpenAI tools 描述生成 |
| `src/tools/PathUtils.*` | 工作区路径解析和越界防护 |
| `src/tools/FileTools.*` | `read_file`、`write_file`、`edit_file`、`list_dir` |
| `src/tools/ShellTool.*` | `run_command` 命令工具 |
| `src/tools/WebSearchTool.*` | `web_search` 和 `/search <query>` 检索结果整理 |
| `tests/*.cpp` | 配置、工具、安全、历史、Skill、MCP、AgentLoop 等单元测试 |

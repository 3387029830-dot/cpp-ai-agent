# cpp-ai-agent —— 项目结构说明

> 当前阶段：M0.5  
> 目的：解释每个目录和配置文件为什么存在，避免项目结构看起来零散。

## 1. 当前结构

```text
cpp-ai-agent/
├── CMakeLists.txt
├── CMakePresets.json
├── vcpkg.json
├── README.md
├── .gitignore
├── .env.example
├── .clang-format
├── config/
│   ├── settings.json
│   └── skills.json
├── docs/
│   ├── 00-项目结构说明.md
│   ├── 01-需求规格说明书.md

│   └── 03-开发计划书.md
├── src/
│   └── main.cpp
├── tests/
│   └── .gitkeep
└── build/
```

## 2. 根目录文件

根目录放的是“项目级配置文件”。这些文件通常不放进子目录，因为对应工具默认会在项目根目录查找它们。

| 文件 | 作用 |
|------|------|
| `CMakeLists.txt` | CMake 的项目构建说明 |
| `CMakePresets.json` | CMake 的预设配置，统一构建命令 |
| `vcpkg.json` | vcpkg 的依赖清单 |
| `README.md` | 项目入口说明 |
| `.gitignore` | Git 忽略规则 |
| `.env.example` | 环境变量样例 |
| `.clang-format` | 代码格式化规则 |

## 3. 子目录

| 目录 | 作用 |
|------|------|
| `config/` | 放程序运行配置，比如模型、权限、Skill 配置 |
| `docs/` | 放项目文档 |
| `src/` | 放 C++ 源代码 |
| `tests/` | 放单元测试代码 |
| `build/` | 放 CMake 生成文件和 exe，可删除重建，不提交 Git |

## 4. 为什么不把所有配置都放进 config/

`config/` 只放“程序运行时自己读取的配置”。

例如：

- `settings.json`
- `skills.json`

而下面这些文件是“开发工具读取的配置”，通常必须或推荐放在根目录：

- `CMakeLists.txt`
- `CMakePresets.json`
- `vcpkg.json`
- `.gitignore`
- `.clang-format`

所以它们不是乱放，而是按工具约定放在根目录。

## 5. 后续结构演进

M1 以后，`src/` 会继续拆分：

```text
src/
├── main.cpp
├── core/
├── llm/
├── tools/
├── security/
├── storage/
└── ui/
```

这些目录会随着功能实现逐步出现，不会在 M0.5 阶段一次性空建太多目录。

# cpp-ai-agent —— 版本日志

## v0.1.0 - 2026-07-06

### 总览

v0.1.0 是第一个完整演示版，覆盖 M0-M5：

- CMake + vcpkg 工程骨架。
- OpenAI-compatible LLM 对话。
- Agent 工具调用主循环。
- 本地文件/命令工具。
- 权限确认和日志回放。
- FTXUI 状态页、诊断命令和演示材料。

### v0.1.0 后本地增强

- `/ui` 从单一状态页调整为 ANSI 流式控制台风格的 Conversation / Tools / Status / Input 概览。
- 主对话循环接入 `src/ui/Console.*`，以语义化样式输出用户消息、助手回复、工具调用、工具结果、权限确认和警告。
- Assistant 回复支持按 UTF-8 字符边界的伪流式打字机输出，非交互输出自动降级为整段打印。
- 新增最小 MCP stdio 客户端，支持 `initialize`、`notifications/initialized`、`tools/list`、`tools/call`。
- `/mcp-demo` 改为启动内置 MCP 测试 server，真实完成 stdio 握手并列出工具。
- 新增 `/mcp-call-demo`，调用内置 MCP 测试 server 的 `echo` 和 `project_info` 工具并显示结果。
- 新增 `/mcp-connect <command> [args...]`，可连接外部 stdio MCP server 并发现工具。
- 新增 `McpToolAdapter`，将内置 MCP 工具注册为 `mcp_echo`、`mcp_project_info`，模型可通过 AgentLoop 自动调用。
- 新增 `ContextManager`，用于保留 system prompt 并限制发送给模型的上下文窗口。
- 新增 `ILlmClient` 接口，让 AgentLoop 可以注入 mock LLM 做稳定单元测试。
- 抽出 `LlmParsing`，单独测试 `tool_calls` 解析逻辑。
- 自动化测试从 18 个 doctest 用例增加到 25 个。

### 已知边界

- `/ui` 是流式控制台概览；项目不接管全屏事件循环，主交互仍保持同步阻塞式行式控制台。
- `/search` 是演示占位。
- MCP 当前覆盖 stdio 工具发现、基础工具调用和内置 MCP 工具的 AgentLoop 自动调用；外部 MCP server 尚未自动注册，也不包含 resources、prompts、SSE/HTTP transport。
- 真实 API 调用需要有效 API Key。
- 当前 HTTP 客户端面向 OpenAI-compatible API。

# cpp-ai-agent —— 版本日志

## v0.2.0 - 2026-07-10

### SSE 流式输出升级

- LLM 客户端从同步 HTTP 升级为 SSE（Server-Sent Events）协议，实现逐 token 实时推送。
- `ILlmClient` 接口重构为单入口设计：仅一个纯虚方法 `chatStream()`。
- `AgentLoop` 新增 `AssistantChunk`/`AssistantDone` 流式事件类型和 `enableStreaming` 开关。
- Console 支持逐字平滑渲染（默认 8ms/字），消除 API burst 推送的卡顿感。

## v0.1.0 - 2026-07-06

### 总览

v0.1.0 是第一个完整演示版，当前文档按 M0-M8 描述：

- CMake + vcpkg 工程骨架。
- OpenAI-compatible LLM 对话。
- Agent 工具调用主循环。
- 本地文件/命令工具。
- 权限确认和日志回放。
- ANSI 流式控制台、诊断命令和演示材料。
- M6 Web 搜索整理。
- M7 MCP 最小客户端。
- M8 项目级规范、跨环境构建、中文命令行参数和双平台同步稳定性。

### v0.1.0 后本地增强

- LLM 客户端从同步 HTTP 升级为 SSE 流式协议：通过 cpr 原生 `ServerSentEventCallback` 解析 `text/event-stream`，实现逐 token 实时推送。
- `ILlmClient` 接口重构为单入口设计：仅一个纯虚方法 `chatStream()`（接收可选 `ChunkCallback`），`chat()` 降级为非虚便利方法，内部调用 `chatStream()`。
- `LlmClient` 提取 `configureSession()` 消除 URL/Header/SSL/Proxy 的重复代码；新增 `ToolCallFragment` 按 index 增量累积流式 tool_calls。
- `AgentLoop` 新增 `enableStreaming` 参数控制 SSE 开关；通过 `AgentEventCallback` 发射 `AssistantChunk`（文本增量）和 `AssistantDone`（流结束）事件，工具调用前先闭合流式块避免输出粘连。
- Console 新增 `printAssistantChunk()` 和 `finishAssistantStream()`，内部管理 `streaming_` 状态自动处理头部/尾部；当 `typewriter_` 启用时，chunk 按 UTF-8 字符边界逐字输出（默认 8ms/字），将 API 的 burst 推送平滑为均匀流。
- `main.cpp` 事件回调改为纯映射：`AssistantChunk` → `printAssistantChunk`、`AssistantDone` → `finishAssistantStream`、`AssistantMessage`（非流式）→ `printAssistant`，零状态泄漏。
- `/ui` 从单一状态页调整为 ANSI 流式控制台风格的 Conversation / Tools / Status / Input 概览。
- 主对话循环接入 `src/ui/Console.*`，以语义化样式输出用户消息、助手回复、工具调用、工具结果、权限确认和警告。
- 主对话新增 slash 命令面板：输入 `/`、`/?`、`/help` 或 `/commands` 可按“主对话命令 / 独立运行命令 / 内部命令”查看当前源码中已注册的所有可执行入口；未知 slash 命令会提示查看命令面板。
- Assistant 回复支持按 UTF-8 字符边界的伪流式打字机输出，非交互输出自动降级为整段打印。
- 新增写入和编辑文件前的 diff 预览确认，`write_file` 和 `edit_file` 会在权限提示中展示待应用变更。
- 权限确认提示改为可选中 UI，支持方向键切换 yes/no 并回车确认；删除文件等危险命令也会进入确认流程。
- 新增最小 MCP stdio 客户端，支持 `initialize`、`notifications/initialized`、`tools/list`、`tools/call`。
- `/mcp-demo` 改为启动内置 MCP 测试 server，真实完成 stdio 握手并列出工具。
- 新增 `/mcp-call-demo`，调用内置 MCP 测试 server 的 `echo` 和 `project_info` 工具并显示结果。
- 新增 `/mcp-connect <command> [args...]`，可连接外部 stdio MCP server 并发现工具。
- 修复 `/load` 历史恢复选择器会显示当前空日志的问题；现在只展示可恢复会话，并为日志文件名加入毫秒和进程 ID，避免同秒启动时文件名冲突。
- 新增 `McpToolAdapter`，将内置 MCP 工具注册为 `mcp_echo`、`mcp_project_info`，模型可通过 AgentLoop 自动调用。
- 新增 `config/mcp_servers.json`，可启用外部 stdio MCP server 并将其工具自动注册为 `mcp_<server>_<tool>`。
- 新增 `ContextManager`，用于保留 system prompt 并限制发送给模型的上下文窗口。
- 新增 `ILlmClient` 接口，让 AgentLoop 可以注入 mock LLM 做稳定单元测试。
- 抽出 `LlmParsing`，单独测试 `tool_calls` 解析逻辑。
- `/skills` 从静态展示扩展为 Skill 能力入口，主对话支持 `/use-skill <name>` 注入专家提示。
- 内置 `code_review`、`cpp_debug`、`project_summary`、`test_writer` 四个 Skill。
- Skill 支持 `allowed_tools` 工具白名单和 `/use-skill <name> [target]` 目标参数，AgentLoop 通过 `ToolPolicy` 回调提供程序级硬拦截，与提示词注入形成双重约束。
- 新增 `/clear-skill` 命令，可在对话中退出当前 Skill 恢复普通对话模式，无需重启程序。
- 新增 `list_dir` 安全工具，支持目录型目标先列文件再调用 `read_file`。
- 新增 `web_search` 安全工具和 `/search <query>` 命令，使用 Bing RSS / DuckDuckGo Instant Answer 返回结构化检索结果、URL 和整理建议。
- 新增 `web_search.proxy_url` / `WEB_SEARCH_PROXY_URL` 专用代理配置，避免 Web 搜索和 LLM API 共用代理设置。
- Skill 默认白名单可按需允许 `web_search`，方便 code review、C++ debug、项目总结和测试生成场景补充外部检索。
- 新增根目录 `AGENTS.md`，启动时自动读取项目级规范并追加到 system prompt，覆盖编码规范、目录约定、构建命令、行为偏好和禁区规则。
- MCP 客户端重构为持久会话：`StdioMcpClient` 新增 `ensureConnected()`，`McpToolAdapter` 复用同一子进程连接，避免每次工具调用都重新启动进程和握手，大幅降低 `npx` 类 MCP server 的调用延迟。
- Web 搜索新增 TTL 5 分钟内存缓存：相同查询在有效期内直接返回缓存结果，避免 Agent 循环中重复发起相同 HTTP 请求。
- 默认 CMake preset 改为 Visual Studio generator，避免 NMake 未加载 MSVC 环境时出现 `cstdint` 等标准库头文件找不到的问题；保留 `nmake-vcpkg-debug` 作为备用 preset。
- 修复 Windows 中文命令行参数解析，`/search "MCP 是什么"` 等中文命令不再显示乱码。
- 自动化测试从 18 个 doctest 用例增加到 36 个。
- 新增 `/load` 命令，可在运行时加载历史会话日志并恢复上下文继续对话。支持 `/load`（交互式编号选择）和 `/load <path>`（直接加载）两种方式。
- 会话日志文件名新增标题后缀：首条用户消息后自动将日志文件重命名为 `session-YYYYMMDD-HHMMSS-<标题>.jsonl`，便于从文件列表中快速识别会话内容。
- `/history` 命令同步显示日志文件标题，`HistoryReader` 新增 `loadMessagesFromHistory()` 函数将 JSONL 日志反序列化为 `Message` 列表。
- 修复 `/load` 加载历史后提问出现 HTTP 400 的问题：根因是 `loadMessagesFromHistory()` 在遇到首个 `tool_result` 时过早 flush Assistant 消息，导致 Agent 多轮工具调用场景中后续 `tool_call`/`tool_result` 成为孤儿、消息角色序列违反 API 约束。修复方案为将 Tool 消息缓冲至下一个 `assistant_message` 或 `user_message` 时统一 flush，保证输出始终为 `Assistant(toolCalls) → Tool → Tool → ...` 的合法顺序。
- LlmClient 流式请求 400 错误新增请求诊断信息（消息数量 + 角色序列），`/load` 中 `std::filesystem::equivalent` 调用增加异常保护。

### 已知边界

- `/ui` 是流式控制台概览；项目不接管全屏事件循环，主交互仍保持同步阻塞式行式控制台。
- `/search <query>` 是轻量 Web 检索入口；依赖网络连通性、可选 `web_search.proxy_url` 代理配置和 Bing / DuckDuckGo 结构化结果返回。
- MCP 当前覆盖 stdio 工具发现、基础工具调用、内置 MCP 工具和配置启用的外部 stdio MCP 工具自动注册；尚不包含 resources、prompts、SSE/HTTP transport。
- 真实 API 调用需要有效 API Key。
- LLM 通信已升级为 SSE 流式协议；MCP 的 SSE/HTTP transport 是独立话题（指 MCP server 连接方式，非 LLM API 调用），尚未实现。

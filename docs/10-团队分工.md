## 4. 组员 A：LLM 与 Agent 主循环

### 负责内容

- `LlmClient`：
  - 读取模型配置。
  - 发送 OpenAI-compatible `/chat/completions` 请求（SSE 流式 + HTTP 双路径）。
  - 解析 assistant 内容和 `tool_calls`（含流式增量累积）。
  - `ILlmClient` 接口设计：单纯虚方法 `chatStream()`。
- `AgentLoop`：
  - 把工具 schema 传给模型。
  - 通过 `AssistantChunk`/`AssistantDone` 事件驱动流式 UI。
  - 分发工具调用。
  - 将工具结果作为 `tool` 消息回填。
  - 最大轮次保护。
- `ContextManager`：
  - 保留 system prompt。
  - 保留最新上下文。
  - 避免上下文无限增长。
- `AGENTS.md`：
  - 程序启动时读取项目级规范。
  - 将编码规范、目录约定、构建命令和行为偏好追加到 system prompt。
  - 区分全局项目规则和 `/use-skill` 临时工作模式。

### 答辩要讲清楚

- 为什么需要 `tool_calls`：模型不直接操作文件，而是请求程序执行工具。
- 为什么工具结果要回填给模型：模型需要基于真实结果生成最终回答。
- 为什么需要最大轮次保护：避免模型无限循环调用工具。
- 配置优先级：系统环境变量 > `.env` > `config/settings.json`。

### 对应测试

- `tests/llm_parsing_tests.cpp`
- `tests/context_manager_tests.cpp`
- `tests/config_tests.cpp`

## 6. 组员 C：交互、Skill、MCP 和 Web 搜索

### 负责内容

- `Console`：
  - ANSI 流式控制台。
  - 用户/助手/工具/权限/警告的分层输出。
  - SSE 驱动的真正流式输出（`printAssistantChunk`/`finishAssistantStream`）。
  - UTF-8 安全的逐字平滑渲染（8ms/字）。
- Skill：
  - `/skills`
  - `/use-skill`
  - `allowed_tools` 白名单限制。
- MCP：
  - `McpClient`
  - `McpToolAdapter`
  - `/mcp-demo`
  - `/mcp-call-demo`
  - `/mcp-connect`
  - `config/mcp_servers.json`
- Web 搜索整理：
  - `/search <query>`
  - 搜索结果摘要和来源展示。

### 答辩要讲清楚

- 为什么选择 ANSI 流式控制台，而不是继续做全屏 FTXUI：同步阻塞架构更适合流式行式输出。
- Skill 的意义：让智能体进入某种工作模式，并限制可用工具范围。
- MCP 的意义：让智能体通过标准协议连接外部工具包。
- 当前 MCP 已做到 stdio 握手、工具发现、基础工具调用和内置工具自动注册；尚未做 SSE/HTTP transport、resources、prompts。
- Web 搜索属于 M6，重点说明 Bing RSS 优先、DuckDuckGo/Bing 兜底链接和代理配置。
- MCP 属于 M7，重点说明“AI 通过标准协议连接外部工具包”。

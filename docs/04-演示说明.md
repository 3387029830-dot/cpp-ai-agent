## 4. 工具调用演示

输入：

```text
请读取 README.md，并用三点总结这个项目。
```

预期结果：

```text
▸ assistant
  我先读取 README.md，然后给你一个简洁的结构化总结。

● read_file  safe
  {"path":"README.md"}
└ read_file: # cpp-ai-agent...

▸ assistant
  这个README.md的核心意思是...
```

说明：这展示了 M3 Agent 主循环，模型通过 SSE 逐 token 流式输出思考过程，然后调用工具，程序执行工具后把结果回填给模型，模型继续流式生成最终回答。`▸ assistant` 后的文字是逐字流式输出的。

## 7. Web 搜索整理演示

直接运行：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /search C++ programming language
```

默认不走代理。如果外网超时或网络需要代理，可以检查 `config/settings.json` 中的 `web_search.proxy_url`，也可以在当前 PowerShell 中临时覆盖：

```powershell
$env:WEB_SEARCH_PROXY_URL="http://127.0.0.1:7897"
.\build\msvc-vcpkg-debug\ai-agent.exe /search C++ programming language
```

预期结果：显示 `Web search results`、`Query`、`Provider`、`Top answer` 或编号结果、URL 和整理建议。

> **结果缓存**：相同查询在 5 分钟内重复搜索会直接返回缓存结果，避免重复 HTTP 请求。在 Agent 循环中模型多次调用同一搜索时尤其有用。

## 8. Web Control Deck 双界面演示

默认启动仍然是终端 TUI/流式控制台。如果希望投屏展示更完整的工作台界面，可以从项目根目录运行：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /web
```

打开浏览器：

```text
http://127.0.0.1:8080
```

也可以指定端口：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /web 8090
.\build\msvc-vcpkg-debug\ai-agent.exe --ui web 8090
```

Web Control Deck 是面向投屏演示的浏览器工作台：

- 左侧 Rail：Chat / Tools / Skills / Runs 导航、工作区信息和快速提示词。
- 中间 Stage：对话流、命令栏、文件拖拽上传区、多行输入框和发送按钮。
- 右侧 Inspector：模型状态、权限模式、工具列表、Skill 列表、命令列表和事件时间线。
- 权限弹窗：`write_file`、`edit_file`、`run_command` 等高风险动作可直接在浏览器中 Approve / Deny。

建议现场输入：

```text
请读取 README.md，总结项目能力，并指出下一步可以演示哪个功能。
```

也可以点击上传按钮或拖入文本文件，文件会保存到项目根目录 `uploads/` 下，方便后续让 Agent 读取和处理。右侧 Permission Mode 可以在 `ask_each_time`、`read_only`、`auto_approve_writes` 之间切换，用来演示不同安全策略。Web Control Deck 和 TUI 使用同一个 `AgentLoop`、工具系统、权限管理和 JSONL 日志，因此它是“界面模式切换”，不是另一套简化 demo。

## 9. Skill 演示

查看 Skill：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /skills
```

主对话中启用 code review Skill：

```text
/use-skill code_review src/mcp
请审查这个目录下 MCP 相关实现的主要风险。
```

预期结果：显示 active skill、target、allowed tools；后续工具调用会被 Skill 白名单限制。

> **双重约束机制**：Skill 的 `allowed_tools` 白名单通过两层防护生效：
> 1. **提示词层**（`SkillCatalog`）：向模型注入 system message，声明 "Do not call tools outside this list"，引导模型自愿遵守。
> 2. **硬拦截层**（`AgentLoop::ToolPolicy`）：在 `main.cpp` 中注册回调，当模型仍尝试调用白名单外的工具时，阻止工具执行并返回错误信息给模型。
>
> 第一层依赖模型配合（不可靠），第二层是程序级强制拦截（可靠）。两者结合保证 Skill 白名单的实际约束力。

清除当前 Skill：

```text
/clear-skill
```

预期结果：显示 `skill> cleared: <name>`，恢复到无 Skill 约束的普通对话模式。

## 10. AGENTS.md 项目规范演示

查看项目级 Agent 规范：

```powershell
type AGENTS.md
```

说明：

- `AGENTS.md` 是全局项目规范，程序启动时自动读取并追加到 system prompt。
- 它适合放编码规范、目录职责、构建命令、测试要求和禁区规则。
- 它和 Skill 不同：`AGENTS.md` 每次启动都生效，Skill 是对话中临时切换的工作模式。

## 11. MCP 演示

内置 MCP 握手：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /mcp-demo
```

内置 MCP 工具调用：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /mcp-call-demo
```

外部 stdio MCP server 连接：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /mcp-connect <command> [args...]
```

> **持久会话**：通过 `McpToolAdapter` 注册进 AgentLoop 的 MCP 工具会维持长连接——首次调用时启动子进程并完成 initialize 握手，后续调用复用同一进程，避免每次工具调用都重新启动（对于 `npx` 等 Node 工具冷启动可节省 500ms-2s）。

## 12. 会话续接演示

启动程序后输入 `/load` 查看可续接的会话列表：

```text
/load
```

预期结果：

```text
load> select a session to resume:

  [1] 2026-07-10 14:30:22   12500 bytes  项目编译问题求助
  [2] 2026-07-10 10:15:00    8200 bytes  如何优化C++性能
  [0] cancel

▸ load
```

输入数字选择要续接的会话（如 `1`），程序会加载历史消息并恢复上下文：

```text
loaded> session-20260710-143022-项目编译问题求助.jsonl (32 messages)
context> 我在编译项目时遇到了模板推导失败的错误...

▸ you
```

之后即可在恢复的上下文中继续对话。也可以直接指定路径：

```text
/load logs\session-20260710-143022-项目编译问题求助.jsonl
```

**日志标题命名**：每次启动后的第一条用户消息会自动截取前 60 个字符，经安全清理后追加到日志文件名中。例如输入 `如何优化C++编译速度？` 后，日志文件自动重命名为 `session-20260710-143022-如何优化C++编译速度.jsonl`。

> **实现细节**：`/load` 从 JSONL 日志中反序列化 User、Assistant、Tool 消息，跳过 System Prompt 和 meta 事件（skill_selected、warning 等），加载后使用当前的 System Prompt（含最新 AGENTS.md），确保项目规范始终是最新版本。

## 13. 讲解重点

- M1：真实 OpenAI 兼容 API 对话，支持 SSE 逐 token 流式输出。
- M2：插件式工具接口、工具注册表、文件和命令工具。
- M3：模型自动发起工具调用，工具结果回填给模型，流式输出思考过程。
- M4：可选中权限确认、diff preview、危险命令拦截、JSONL 日志和历史回放。
- M5：ANSI 流式控制台（含逐字平滑渲染）、Skill、配置诊断、Provider 样例和演示命令。
- M6：Web 搜索整理，国内网络优先 Bing RSS，提供 DuckDuckGo/Bing 兜底链接，内置 TTL 5 分钟结果缓存。
- M7：MCP 客户端，覆盖 stdio 握手、工具发现、工具调用和 AgentLoop 工具适配，支持持久会话复用子进程。
- M8：工程化稳定性，包括 `AGENTS.md` 项目规范注入、Visual Studio generator 默认构建、NMake 备用方案、固定 exe 输出目录和 Windows 中文命令行参数修复。

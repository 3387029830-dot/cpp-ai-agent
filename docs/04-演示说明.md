# cpp-ai-agent —— 演示说明

> 目标：用最短路径展示项目从基础对话到工具调用、权限控制、日志回放、Skill、Web 搜索、MCP 和跨环境稳定性的 M0-M8 完整闭环。

## 1. 构建和测试

```powershell
cmake --preset msvc-vcpkg-debug
cmake --build --preset msvc-vcpkg-debug
cd build\msvc-vcpkg-debug
ctest -C Debug --output-on-failure
cd ..\..
```

预期结果：

```text
100% tests passed
```

## 2. 配置检查

查看当前实际读取到的配置：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /config
```

运行本地诊断：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /doctor
```

如果使用 DeepSeek，可以生成 `.env` 模板：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /init-env deepseek
notepad .env
```

## 3. 基础对话

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe
```

输入：

```text
你好，请用一句话介绍你自己。
```

预期结果：模型返回自然语言回答。

## 4. 工具调用演示

输入：

```text
请读取 README.md，并用三点总结这个项目。
```

预期结果：

```text
tool_call> read_file {"path":"README.md"}
tool_result> read_file: ...
assistant> ...
```

说明：这展示了 M3 Agent 主循环，模型请求工具，程序执行工具，再把结果回填给模型。

## 5. 目录列表演示

输入：

```text
请列出 src/agent 下有哪些文件。
```

预期结果：

```text
tool_call> list_dir {"path":"src/agent"}
tool_result> list_dir: ...
assistant> ...
```

说明：目录目标应先使用 `list_dir`，再按需要逐个读取具体文件。

## 6. 权限控制演示

输入：

```text
请创建 temp-demo.txt，内容是 hello agent
```

预期结果：

```text
! permission required
  tool: write_file  risk: write
  diff preview
  ...
  select yes/no with arrow keys, press Enter
```

用方向键选中 `yes` 并回车后，程序才会写入文件。

## 7. Web 搜索整理演示

直接运行：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /search C++ programming language
```

默认不走代理。如果外网超时或网络需要代理，可以检查 `config/settings.json` 中的 `web_search.proxy_url`，也可以在当前 PowerShell 中临时覆盖：

```powershell
$env:WEB_SEARCH_PROXY_URL="http://127.0.0.1:7897"
.\build\msvc-vcpkg-debug\ai-agent.exe /search C++ programming language
```

预期结果：显示 `Web search results`、`Query`、`Provider`、`Top answer` 或编号结果、URL 和整理建议。

## 8. Skill 演示

查看 Skill：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /skills
```

主对话中启用 code review Skill：

```text
/use-skill code_review src/mcp
请审查这个目录下 MCP 相关实现的主要风险。
```

预期结果：显示 active skill、target、allowed tools；后续工具调用会被 Skill 白名单限制。

## 9. MCP 演示

内置 MCP 握手：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /mcp-demo
```

内置 MCP 工具调用：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /mcp-call-demo
```

外部 stdio MCP server 连接：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /mcp-connect <command> [args...]
```

## 10. 历史回放演示

列出日志：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /history
```

回放某次会话：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /replay logs\session-xxxx.jsonl
```

预期结果：按 `user>`、`tool_call>`、`tool_result>`、`assistant>` 回放关键事件。

## 11. 讲解重点

- M1：真实 OpenAI 兼容 API 对话。
- M2：插件式工具接口、工具注册表、文件和命令工具。
- M3：模型自动发起工具调用，工具结果回填给模型。
- M4：可选中权限确认、diff preview、危险命令拦截、JSONL 日志和历史回放。
- M5：ANSI 流式控制台、Skill、配置诊断、Provider 样例和演示命令。
- M6：Web 搜索整理，国内网络优先 Bing RSS，并提供 DuckDuckGo/Bing 兜底链接。
- M7：MCP 最小客户端，覆盖 stdio 握手、工具发现、工具调用和 AgentLoop 工具适配。
- M8：工程化稳定性，包括 Visual Studio generator 默认构建、NMake 备用方案、固定 exe 输出目录和 Windows 中文命令行参数修复。

# cpp-ai-agent —— 答辩提纲

## 1. 项目定位

`cpp-ai-agent` 是一个使用 C++17 实现的终端 AI 编程智能体。项目目标不是复刻完整商业 Coding Agent，而是展示一个可运行、可扩展、可测试的最小闭环。

## 2. 核心能力

- 接入 OpenAI 兼容 API，支持 LinkAPI、DeepSeek 等 provider 配置。
- 支持多轮 Session 上下文。
- 支持模型自动发起工具调用。
- 支持文件读取、写入、局部编辑和命令执行工具。
- 支持权限确认和危险命令拦截。
- 支持 JSONL 日志记录和历史回放。
- 支持配置诊断、Provider 模板、FTXUI 状态页、Skill 展示和 mock MCP 演示。

## 3. 架构讲解

```text
Console / FTXUI Status
        |
    AgentLoop
        |
        +-- LlmClient
        +-- ToolRegistry
        +-- PermissionManager
        +-- JsonLogger / HistoryReader
```

讲解重点：

- `LlmClient` 负责 OpenAI-compatible 请求和 `tool_calls` 解析。
- `ToolRegistry` 负责工具注册、查找和 tools schema 生成。
- `AgentLoop` 负责模型、工具和会话之间的闭环。
- `PermissionManager` 把工具风险分成 Safe、Write、Dangerous。
- `JsonLogger` 与 `HistoryReader` 让执行过程可追溯。

## 4. 演示顺序

1. `/doctor` 展示配置诊断。
2. 普通对话展示 M1。
3. “读取 README.md 并总结”展示 M3 工具调用。
4. “创建 temp-demo.txt”展示 M4 权限确认。
5. `/history` 和 `/replay` 展示日志回放。
6. `/ui` 展示 FTXUI 状态页。
7. `/skills`、`/search`、`/mcp-demo` 展示 M5 扩展边界。

## 5. 测试说明

```powershell
cmake --build --preset msvc-vcpkg-debug
cd build\msvc-vcpkg-debug
ctest --output-on-failure
```

当前测试覆盖：

- 配置读取和 `.env` 优先级。
- API 配置诊断。
- 工具注册和文件工具。
- 路径越界防护。
- 权限确认逻辑。
- 历史日志列表和回放。

## 6. 代码质量

- 使用 `.clang-format` 统一 C++ 风格。
- MSVC 构建使用 `/W4 /permissive-`。
- 非 MSVC 构建使用 `-Wall -Wextra -Wpedantic`。
- 密钥不进入仓库，真实配置放 `.env` 或系统环境变量。
- `.env`、`logs/`、`build/`、`vcpkg_installed/` 均已忽略。

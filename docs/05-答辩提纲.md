# cpp-ai-agent —— 答辩提纲

## 1. 项目定位

`cpp-ai-agent` 是一个使用 C++17 实现的终端 AI 编程智能体。项目目标不是复刻完整商业 Coding Agent，而是展示一个可运行、可扩展、可测试的最小闭环。

## 2. 核心能力

- 接入 OpenAI 兼容 API，支持 LinkAPI、DeepSeek 等 provider 配置。
- 支持多轮 Session 上下文。
- 支持模型自动发起工具调用。
- 支持目录列表、文件读取、写入、局部编辑、命令执行和 Web 搜索整理工具。
- 支持可选中权限确认、diff 预览和危险命令拦截。
- 支持 JSONL 日志记录和历史回放。
- 支持配置诊断、Provider 模板、ANSI 流式控制台、Skill 激活、MCP stdio 客户端和外部 MCP 工具发现。

## 2.1 里程碑讲法

答辩时可以把项目分成三层：

- M0-M5：核心智能体闭环。从工程骨架、模型对话、工具系统、Agent 主循环、安全日志，到最终可演示的控制台和文档。
- M6：联网搜索扩展。新增 `web_search` 和 `/search <query>`，优先使用 Bing RSS，失败时给出 DuckDuckGo/Bing 兜底链接，内置 TTL 5 分钟结果缓存。
- M7：MCP 最小客户端。实现 stdio MCP 握手、工具发现、持久会话连接和基础工具调用，避免每次调用重启子进程；通过 `McpToolAdapter` 把 MCP 工具适配进 AgentLoop。
- M8：工程化稳定性。解决项目级 Agent 规范、新电脑构建、NMake 环境、固定 exe 输出、Windows 中文命令行参数和 GitHub/GitLab 同步问题。

总负责人可以这样概括：

> 我们最初完成 M0-M5 交付版，保证智能体主线可运行、可测试、可演示；后续又扩展了 M6 联网搜索、M7 MCP 最小客户端和 M8 跨环境工程化稳定性，让项目更接近真实可落地的 AI Agent。

## 3. 架构讲解

```text
ANSI Console UI
        |
    AgentLoop
        |
        +-- LlmClient
        +-- ToolRegistry
        +-- PermissionManager
        +-- JsonLogger / HistoryReader
        +-- SkillCatalog
        +-- McpClient / McpToolAdapter
        +-- AGENTS.md project rules
```

讲解重点：

- `LlmClient` 负责 OpenAI-compatible 请求和 `tool_calls` 解析。
- `ToolRegistry` 负责工具注册、查找和 tools schema 生成。
- `AgentLoop` 负责模型、工具和会话之间的闭环。
- `PermissionManager` 把工具风险分成 Safe、Write、Dangerous，并驱动 diff 预览确认。
- `JsonLogger` 与 `HistoryReader` 让执行过程可追溯。
- `SkillCatalog` 通过提示词注入和 `allowed_tools` 形成可切换工作模式；`AgentLoop::ToolPolicy` 提供程序级硬拦截，与提示词层形成双重约束，确保白名单工具限制不被模型绕过。
- `McpClient` 与 `McpToolAdapter` 负责 MCP stdio 握手、工具发现和工具调用适配。
- `AGENTS.md` 在启动时自动读取，作为项目级规范追加到 system prompt；它约束整个会话，Skill 则用于临时任务模式。

## 4. 演示顺序

1. `/doctor` 展示配置诊断。
2. 普通对话展示 M1。
3. “读取 README.md 并总结”展示 M3 工具调用。
4. “列出 src/agent 下的文件”展示 `list_dir` 目录工具。
5. “创建 temp-demo.txt”展示 M4 diff 预览和可选中权限确认。
6. `/search C++ programming language` 展示 Web 搜索整理。
7. `/skills` 和 `/use-skill code_review src/mcp` 展示 Skill 激活和工具白名单；`/clear-skill` 展示退出 Skill 恢复普通对话。
8. `/mcp-demo`、`/mcp-call-demo` 展示 MCP stdio 握手和工具调用。
9. `/history` 和 `/replay` 展示日志回放。
10. `/ui` 展示 ANSI 流式控制台概览。
11. 展示 `AGENTS.md`，说明项目级规范如何注入 system prompt。
12. 说明 M8：默认使用 Visual Studio generator，避免新电脑 NMake 环境缺失导致 `cstdint` 找不到；中文命令如 `/search "MCP 是什么"` 已正常支持。

## 5. 测试说明

```powershell
cmake --build --preset msvc-vcpkg-debug
cd build\msvc-vcpkg-debug
ctest -C Debug --output-on-failure
```

当前测试覆盖：

- 配置读取和 `.env` 优先级。
- API 配置诊断。
- 工具注册和文件工具。
- 路径越界防护。
- 权限确认逻辑。
- Skill 配置和工具白名单。
- AGENTS.md 项目规范加载。
- MCP 请求生成、响应解析和持久会话。
- Web 搜索结果格式化与缓存。
- 历史日志列表和回放。

## 6. 代码质量

- 使用 `.clang-format` 统一 C++ 风格。
- MSVC 构建使用 `/W4 /permissive-`。
- 非 MSVC 构建使用 `-Wall -Wextra -Wpedantic`。
- 密钥不进入仓库，真实配置放 `.env` 或系统环境变量。
- `.env`、`logs/`、`build/`、`vcpkg_installed/` 均已忽略。

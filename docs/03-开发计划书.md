# AI 编程智能体 —— 开发计划书

> 项目代号：cpp-ai-agent  
> 版本：v1.2
> 日期：2026-07-07

## 1. 开发原则

本项目采用“先跑通主线，再逐步增强”的方式开发。

每个阶段都必须满足：

- 有明确目标。
- 有可运行产物。
- 有验收方式。
- 出现问题时可以回退到上一阶段。

## 2. 里程碑总览

| 阶段 | 名称 | 核心产出 | 验收标准 |
|------|------|----------|----------|
| M0 | 环境与骨架 | CMake 项目、最小可运行程序 | 能配置、编译、运行 |
| M0.5 | 工程脚手架补齐 | vcpkg、配置文件、测试目录、格式化配置 | 能通过 preset 构建并使用依赖 |
| M1 | 模型对话 | LlmClient、Session、基础输入输出 | 能真实调用模型 |
| M2 | 工具系统 | ITool、ToolRegistry、FileTools、ShellTool | 工具可独立测试 |
| M3 | Agent 主循环 | 模型自动调用工具并回填结果 | 能完成“读文件并总结” |
| M4 | 安全与日志 | PermissionManager、Logger、历史回放 | 高危操作确认，日志可读 |
| M5 | 完善交付 | ANSI 流式控制台、配置诊断、Skill、演示材料 | 可完整答辩演示 |
| M6 | 联网搜索 | Web 搜索工具、`/search`、Bing RSS 优先、失败兜底链接 | 能查询并整理外部信息 |
| M7 | MCP 最小客户端 | stdio MCP 初始化、工具发现、工具调用、工具适配 | 能连接内置或外部 MCP 工具 |
| M8 | 工程化稳定 | AGENTS.md 项目规范、Visual Studio generator、跨机器构建、中文参数修复、双平台同步 | 新电脑可复现构建和演示 |

## 7. M3：Agent 主循环

目标：模型可以自主决定调用工具。

任务：

- [x] 生成 OpenAI function-calling 格式的 `tools` 定义。
- [x] 解析模型返回的 `tool_calls`。
- [x] 根据工具名分发到 `ToolRegistry`。
- [x] 工具结果回填为 `tool` 消息。
- [x] 设置最大循环次数。
- [x] 把关键事件推送给 UI。

验收：

```text
用户输入：读取 README.md 并总结
模型行为：调用 read_file
程序行为：读取文件并回填结果
最终输出：模型生成总结
```

当前结果：M3 主循环已完成并持续增强。程序启动时注册本地文件工具、命令工具、Web 搜索工具和 MCP 适配工具，模型可以自动请求工具调用，程序执行后将结果以 `tool` 消息回填给模型，再由模型生成最终回答。当前还支持 Skill 工具白名单，避免特定工作模式调用无关工具。

## 8. M4：安全与日志

目标：让系统可控、可追溯。

任务：

- [x] 实现 `PermissionManager`。
- [x] 高风险操作前显示确认信息。
- [x] 实现 JSON Logger。
- [x] 记录用户输入、模型回复、工具调用和结果。
- [x] 实现历史加载和回放。

验收：

- 执行命令前需要确认。
- 拒绝确认后工具不执行。
- 一次完整会话能保存为 JSON。
- 日志不包含完整 API Key。

当前结果：M4 已完成并增强。`read_file`、`list_dir`、`web_search` 可直接执行，`write_file`、`edit_file`、`run_command` 会根据 `permission.mode` 进行确认；写入和编辑会在确认前展示 diff preview，交互式终端支持方向键选择 yes/no 并回车执行。明显危险命令会进入确认或被策略拦截。会话日志以 JSONL 写入 `logs/`，记录用户输入、模型回复、工具调用和工具结果。可通过 `/history` 列出日志，通过 `/replay <日志路径>` 回放历史。

## 13. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 依赖下载失败 | 无法构建 FTXUI/cpr | 提供控制台降级方案，或改用 vcpkg |
| API 不兼容 function-calling | 工具调用失败 | 优先选择 OpenAI 兼容供应商 |
| C++ HTTP/JSON 解析复杂 | 开发变慢 | 同步请求优先，先跑通主线 |
| 命令执行风险 | 误操作本地环境 | 默认确认，危险命令可屏蔽 |
| Web 搜索网络不可用 | `/search` 无法返回上游结果 | 支持 `web_search.proxy_url` 和 `WEB_SEARCH_PROXY_URL` 专用代理配置，失败时输出清晰错误 |
| 时间不足 | 扩展功能做不完 | P0 优先，P1/P2 保持可演示的最小闭环 |

# FORGE-17 工作流模式与专家模式说明

## 目标

工作流模式和专家模式用于把 FORGE-17 从“通用对话智能体”升级成“可切换任务形态的智能体”。

- 工作流模式：注入一套执行步骤，例如先分析、再计划、再执行、最后验证。
- 专家模式：把一个专家提示词和一个 Skill 打包在一起，一键注入当前会话。

这两个能力都可以在 Web Control Deck 的 `Modes` 页面直接点击卡片启用。

## 工作流模式

配置文件：`config/modes.json`

当前内置工作流：

- `plan-execute-verify`：先分析，再制定短计划，执行最小改动，最后构建或测试验证。
- `demo-story`：把零散功能整理成适合答辩展示的清晰叙事。
- `docs-sync`：修改功能时同步更新文档和演示命令。

命令行用法：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe
/modes
/use-workflow plan-execute-verify src/web
/clear-mode
```

## 专家模式

专家模式会同时注入：

1. 专家身份提示词
2. 绑定的 Skill
3. 对应的工具白名单约束

当前内置专家：

- `cpp-architect`：绑定 `cpp_debug`，用于 C++17、CMake、vcpkg、MSVC 问题。
- `ui-deck`：绑定 `ui_design`，用于 Web 控制台和交互设计。
- `mcp-specialist`：绑定 `mcp_protocol`，用于 MCP 协议和外部工具接入。
- `docs-lead`：绑定 `project_summary`，用于项目总结、答辩叙事和文档整理。

命令行用法：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe
/modes
/use-expert cpp-architect CMakePresets.json
/use-expert mcp-specialist src/mcp
/clear-mode
```

## Web 演示方式

启动 Web 模式：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /web
```

浏览器打开：

```text
http://127.0.0.1:8080
```

演示路径：

1. 进入左侧 `Modes` 页面。
2. 点击一个 Workflow mode，例如 `plan-execute-verify`。
3. 右侧 `Mode Injection` 面板会显示当前激活的 Workflow。
4. 再点击一个 Expert mode，例如 `cpp-architect`。
5. 右侧会显示当前 Expert 和绑定 Skill。
6. 在输入框里提出任务，例如“检查当前 Windows 构建配置是否完整”。
7. 需要退出模式时，点击右侧 `Clear`。

## 答辩讲法

可以这样解释：

> 工作流模式解决“智能体做事没有流程”的问题；专家模式解决“智能体没有领域身份”的问题。我们把执行流程、专家提示词和 Skill 工具白名单打包成可注入 preset，让 Web 界面可以一键把通用 Agent 切换成 C++ 架构专家、MCP 协议专家或答辩文档专家。

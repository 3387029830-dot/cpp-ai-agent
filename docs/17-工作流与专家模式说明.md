# FORGE-17 工作流与专家模式说明

## 目标

工作流模式和专家模式用于把 FORGE-17 从“通用对话智能体”升级成“可切换任务形态的智能体”。
- 工作流模式：注入一套执行步骤，例如先分析、再计划、再执行、最后验证。
- 专家模式：把一个专家提示词和一个 Skill 打包在一起，一键注入当前会话。

这两类能力都可以在 Web Control Deck 的 `Modes` 页面直接点击启用。

## 工作流模式

配置文件：`config/modes.json`

当前内置工作流：
- `plan-execute-verify`：先分析，再给短计划，然后最小改动执行，最后构建或测试验证。
- `demo-story`：把零散功能整理成适合答辩演示的叙事主线。
- `docs-sync`：修改功能时同步更新代码、文档和演示命令。

命令行示例：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe
/modes
/use-workflow plan-execute-verify src/web
/clear-mode
```

## 专家模式

专家模式会同时注入：
1. 专家身份提示词
2. 绑定的 Skill
3. 对应的工具白名单

当前内置专家：
- `cpp-architect`：绑定 `cpp_debug`，用于 C++17、CMake、vcpkg、MSVC 问题。
- `ui-deck`：绑定 `ui_design`，用于 Web 控制台和交互设计。
- `mcp-specialist`：绑定 `mcp_protocol`，用于 MCP 协议和外部工具接入。
- `docs-lead`：绑定 `project_summary`，用于项目总结、答辩叙事和文档整理。
- `product-manager`：绑定 `product_strategy`，用于需求分析、优先级、取舍和发布节奏。

命令行示例：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe
/modes
/use-expert product-manager 当前版本要怎么做得更像一个产品
/clear-mode
```

## Web 演示

启动 Web 模式：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe /web
```

浏览器地址：

```text
http://127.0.0.1:8080
```

演示路径：
1. 打开左侧 `Modes`。
2. 先点一个 Workflow mode，例如 `demo-story`。
3. 再点一个 Expert mode，例如 `product-manager`。
4. 右侧会显示当前激活的 workflow、expert 和绑定 skill。
5. 在输入框里提出具体任务，例如“从产品经理视角评估当前界面还缺什么”。
6. 需要退出时点 `Clear`。

## 答辩话术

可以这样解释：
> 工作流模式解决的是“智能体做事没有流程”的问题，专家模式解决的是“智能体没有领域视角”的问题。我们把执行流程、专家提示词和 Skill 工具白名单打包成可注入预设，让 Web 界面可以一键切换成产品经理、C++ 架构师、MCP 专家或文档负责人。




## 新增工作流

- `defense-qa-brief`：把项目事实整理成答辩可直接使用的结论、证据、演示动作和追问应对。

示例：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe
/modes
/use-workflow defense-qa-brief 当前项目怎么讲最能打动评委
```
## 新增工作流

- `boundary-contract`：把模糊需求收敛成任务合同，先明确目标、范围内、范围外、成功标准和风险，再开始执行。

适合场景：
- 需求刚说了一半，需要先问清边界。
- 演示时想突出“人类定义上限，模型在边界内完成任务”。
- 需要避免模型一上来就跑偏、扩写、过度实现。

在 Web 右侧状态栏里，`任务合同` 面板可以直接编辑合同字段、锁定合同，并把当前目标注入到对话中，便于演示“先定边界，再执行”的收敛过程。

示例：

```powershell
.\build\msvc-vcpkg-debug\ai-agent.exe
/modes
/use-workflow boundary-contract 先把这个需求变成合同再开始做
```

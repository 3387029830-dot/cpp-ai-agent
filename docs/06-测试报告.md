## 5. 手动验收项

| 功能 | 验收命令或输入 | 预期结果 |
|------|----------------|----------|
| 配置诊断 | `ai-agent.exe /doctor` | 显示 base_url、model、key、vcpkg、workspace 状态 |
| 基础对话 | `你好` | 模型返回自然语言回复 |
| 文件读取工具 | `请读取 README.md，并用三点总结这个项目。` | 出现 `tool_call> read_file` 和最终总结 |

| 历史回放 | `ai-agent.exe /replay logs\session-xxx.jsonl` | 回放 user/tool/assistant 事件 |
| ANSI 流式控制台概览 | `ai-agent.exe /ui` | 使用同一套 Console 样式显示 Conversation / Tools / Status / Input 概览 |
| Skill 展示 | `ai-agent.exe /skills` | 显示 `config/skills.json` 中的 Skill |
| mock MCP | `ai-agent.exe /mcp-demo` | 显示 mock MCP 调用边界 |

## 6. 已知边界

- `/ui` 是 ANSI 流式控制台概览；主交互不接管全屏，仍是同步阻塞的行式控制台。
- `/search` 是 Web 搜索扩展占位，不会真实联网搜索。
- `/mcp-demo` 是 mock 演示，不包含真实 MCP transport。
- Anthropic-compatible `/anthropic` 接口尚未接入；当前支持 OpenAI-compatible `/chat/completions`。
- 真实 API 验证依赖用户自己的有效 API Key。

## 7. 结论

v0.1.0 已满足课程演示版要求：可构建、可运行、可测试、可调用模型、可执行工具、可确认高风险操作、可记录和回放历史，并具备清晰的扩展边界说明。

# cpp-ai-agent —— 项目总结

> 版本：v0.1.0  
> 日期：2026-07-06

## 1. 项目概述

`cpp-ai-agent` 是一个使用 C++17 实现的终端 AI 编程智能体。项目通过 OpenAI 兼容 API 连接大语言模型，并通过本地工具完成文件读取、写入、局部编辑、命令执行和 Web 搜索整理等任务。

项目面向课程实践和答辩演示，重点展示从需求、设计、编码、测试到交付的完整工程过程。

## 3. 核心设计

项目采用分层设计：

- `core`：消息、角色、会话。
- `core/ContextManager`：构建发送给模型的上下文窗口，保留 system prompt 和最新消息。
- `llm`：OpenAI-compatible API 请求、`tool_calls` 解析。
- `tools`：统一工具接口、注册表、文件、命令和 Web 搜索工具。
- `agent`：Agent 主循环，负责任务编排。
- `security`：权限确认和危险命令拦截。
- `storage`：JSONL 日志和历史回放。
- `diagnostics`：配置诊断和交付排错。
- `ui`：ANSI 增强流式控制台呈现，负责对话、工具事件、权限提示和状态概览输出。
- `mcp`：最小 MCP stdio 客户端，负责启动 server、完成初始化、发现工具列表、调用基础工具，并将内置 MCP 工具适配为 AgentLoop 可调用的 `ITool`。

## 4. 交付能力

当前版本支持：

- 多轮终端对话。
- `.env`、系统环境变量、`settings.json` 多级配置。
- LinkAPI 和 DeepSeek 等 OpenAI-compatible provider。
- 模型自动调用 `read_file` 等工具。
- 模型可自动调用内置 MCP 工具，例如 `mcp_project_info`。
- 写入、编辑前显示 diff 预览并进行权限确认；命令执行前进行权限确认。
- JSONL 会话日志和历史回放。
- ANSI 增强流式控制台：启动横幅、用户/助手消息、工具调用、工具结果、权限提示和 `/ui` 概览。
- `/doctor` 配置诊断。
- `/demo` 演示脚本。
- `/skills`、`/search <query>`、`/mcp-demo` 扩展演示。
- `web_search` 安全工具可由模型在 AgentLoop 中自动调用，返回结构化检索结果、URL 和整理建议。
- `/mcp-call-demo` 内置 MCP 工具调用演示。
- `/mcp-connect <command> [args...]` 外部 stdio MCP server 工具发现。
- 内置 MCP 工具注册进 AgentLoop，支持模型自动工具调用。
- `config/mcp_servers.json` 外部 stdio MCP server 自动发现和工具注册。
- 根目录 `AGENTS.md` 项目级规范自动注入 system prompt，使智能体默认遵守项目编码规范、目录约定、构建命令和禁区规则。
- AgentLoop mock LLM 自动化测试，覆盖工具调用分发、结果回填、最大轮次保护和上下文窗口。

## 5. 安全策略

- API Key 不进入仓库。
- `.env` 被 `.gitignore` 忽略。
- 文件路径限制在 workspace 内。
- 工具风险分级为 Safe、Write、Dangerous。
- Write/Dangerous 工具根据配置进行确认，文件写入和编辑会在确认前展示 diff 预览。
- 明显危险命令会被拦截。

## 6. 后续方向

- 在不引入全屏事件循环的前提下，继续增强流式控制台的可读性和演示效果。
- 为 Web 搜索结果增加缓存、来源评分和更细的引用整理。
- 扩展 MCP：支持更稳定的外部 server 生命周期管理、resources、prompts、SSE/HTTP transport。
- 支持 Anthropic-compatible API。
- 增加更细粒度日志脱敏和命令白名单。

## 7. 总结

v0.1.0 已完成一个可演示 AI Coding Agent 的 M0-M8 闭环。M0-M5 保证主线智能体可运行、可测试、可演示；M6-M8 继续补齐联网搜索、MCP 最小客户端、AGENTS.md 项目规范和跨环境稳定性。它具备清晰的模块边界、可测试的核心模块、可复现的构建流程和可展示的答辩材料，适合作为课程项目最终交付版本。

*   **具体完成任务列表：**
    *   **工具系统设计与实现**：实现 `ITool` 插件式工具接口和 `ToolRegistry` 工具注册表，完成 5 个本地工具——`read_file`（文件读取）、`write_file`（文件写入）、`edit_file`（文件编辑）、`list_dir`（目录列表）、`run_command`（命令执行），支持生成 OpenAI tools schema 供模型调用。
    *   **安全控制模块**：实现 Safe / Write / Dangerous 三级风险分级机制；写入和命令执行前弹出权限确认提示（支持方向键选择 yes/no + 回车确认）；`write_file` 和 `edit_file` 操作前展示 diff 预览，让用户在写入前看到待应用变更；实现 `PathUtils::resolveWorkspacePath()` 路径沙箱，阻止模型访问 workspace 以外的文件系统路径；危险命令（如 `rm -rf`、`format` 等）自动拦截。
    *   **日志与历史回放**：实现 `JsonLogger`，将每轮对话（用户消息、助手回复、工具调用、工具结果、权限决策）以 JSONL 格式完整记录；实现 `/history` 命令展示历史会话列表及自动标题命名（首条用户消息后自动重命名为 `session-YYYYMMDD-HHMMSS-<标题>.jsonl`）；实现 `/replay <path>` 命令完整回放历史会话；实现 `/load <path>` 命令加载历史会话并恢复上下文继续对话，支持 `/load`（交互式编号选择）和 `/load <path>`（直接加载）两种方式。
    *   **项目整体架构 XMind 思维导图**：使用 XMind 制作了项目整体架构思维导图（"AI编程智能体"），覆盖项目定位、运行闭环、Agent 主流程、核心模块（src 下 11 个子模块）、用户交互体验、MCP 集成、配置与依赖、测试体系、答辩演示路径、交付同步和后续扩展共 12 个维度；同时导出了 PDF 版本和配套的架构详解文档（智能体项目整体架构.md/docx），以通俗易懂的"机器人器官"类比方式逐模块解释核心设计思想，方便团队成员和答辩评委快速理解项目全貌。
    *   **AGENTS.md 项目规范注入**：在 `src/main.cpp` 中实现启动时自动读取根目录 `AGENTS.md` 文件，将项目级编码规范（C++17 约束、命名约定、头文件规范）、目录结构约定、构建命令、代码风格要求、模块依赖规则和禁区规则追加到 system prompt 中，让 Agent 在每次对话中都遵循项目规范。该功能区分于 `/use-skill` 的临时工作模式，属于全局项目规则。
    *   **System Prompt 主动追问规则**：在默认 system prompt 中增加了主动追问规则，使 Agent 在需求不明确时能够主动向用户澄清，提升了对话质量和任务完成准确率。
    *   **修复用户输入重复打印**：定位并修复了 `printUser` 函数导致用户输入内容在终端中重复显示的 bug，优化了控制台输出体验。
    *   **修复 web_search GBK 编码崩溃**：在 `fix/encoding-gbk-to-utf8` 分支上修复了 `web_search` 工具返回 GBK 编码中文内容时导致 JSON 解析崩溃的问题。根因是 Windows 环境下部分中文网站返回 GBK 编码响应，而 JSON 解析器期望 UTF-8 输入。修复方案为在解析前检测编码并转换为 UTF-8。

**四、 遇到的问题与困难**

*   **技术问题：**
    *   **web_search GBK 编码兼容性问题**：`web_search` 工具在抓取部分中文网站时，返回 GBK 编码内容导致 `nlohmann/json` 在解析时崩溃。排查过程中发现 Windows 平台的默认代码页（CP936/GBK）与 JSON 库期望的 UTF-8 之间存在编码不匹配。
    *   **已尝试的解决方案：** 查阅了 Windows 编码转换文档（`MultiByteToWideChar` / `WideCharToMultiByte`），在 `src/utils/Encoding.cpp` 中实现编码检测和转换逻辑，将 GBK 响应统一转为 UTF-8 后再交给 JSON 解析器处理。
    *   **需要帮助或待解决：** 无，已独立解决。
*   **非技术问题：**
    *   **时间管理**：刚开始缺少经验，在时间管理上出现问题，进度推进不合理。
    *   **已尝试的解决方案：** 在工作前根据思维导图理清大致工作框架，并对任务进行规划和安排，更好的统筹协调时间，并按计划有条不紊地推进工作。

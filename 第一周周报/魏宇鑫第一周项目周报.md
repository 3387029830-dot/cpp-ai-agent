**四、 遇到的问题与困难**

*   **技术问题：**
    *   **`/load` 后 API 返回 HTTP 400**：加载历史会话后提问时，LLM API 返回 400 错误。通过新增的诊断信息（messages 数量 + roles 序列）发现消息角色序列中存在 `tool → assistant` 的非法顺序。根因是 `loadMessagesFromHistory()` 在遇到第一个 `tool_result` 时就 flush 了 pending Assistant，而 Agent 多轮工具调用场景中同一个 Assistant 可能发起多个工具调用，导致后续 `tool_call`/`tool_result` 成为孤儿消息。修复方案为将 Tool 消息缓冲起来，仅在遇到下一个 `assistant_message` 或 `user_message` 时才统一 flush。
    *   **已尝试的解决方案：** 重构 `loadMessagesFromHistory` 的 flush 策略，新增 `pendingToolMessages` 缓冲队列。
    *   **需要帮助或待解决：** 无

*   **非技术问题：**
    *   **GitHub/GitLab 多仓库同步冲突**：本周涉及 GitHub 和 GitLab 双平台同步，多次 merge 引发冲突（斜杠命令面板合并时与 `/load` 处理器存在代码重复）。已通过精简重构解决。
    *   **已采取的措施：** 将 `/load` 处理器从 ~80 行重复分支精简为 ~40 行统一逻辑，降低后续合并冲突风险。

* **例子：System Prompt 自驱动能力注入**

  ### 方案设计

  指导老师反馈项目"功能完整但零散，没有体现 Agent 自驱动能力"。经分析确认：AgentLoop 的 for 循环天生支持多步工具链（LLM → 工具 → 结果 → LLM，直到空 toolCalls），代码无需改动。问题在 System Prompt——没有教 Agent「收到复杂任务时应该怎么思考」。

  放弃 Skill 方案（为每种任务写 `ppt_creator`、`game_developer`——换任务就打回原形），改为注入**通用自主行为方法论**，适用于 PPT、网页、游戏、歌曲等任何任务。

  详细方案文档：`docs/12-自驱动能力增强方案.md`（v2.0）。

  ### 实现

  在 `main.cpp` System Prompt 中新增 `=== AUTONOMOUS BEHAVIOR METHODOLOGY ===` 章节（~50 行）：

  | # | 规则 | 关键行为 |
  |---|------|----------|
  | 1 | UNDERSTAND | 模糊时反问 2-4 个关键问题，提供选项不假设 |
  | 2 | PLAN | 拆任务为步骤，展示计划给用户确认 |
  | 3 | PREPARE | 检查依赖（pip list / node --version），缺啥自动装 |
  | 4 | RESEARCH | 涉及外部知识主动 web_search，标注来源 URL |
  | 5 | EXECUTE | 逐步执行，每步汇报「✅ Step X/N done」 |
  | 6 | SELF-VERIFY | 产物后 read_file 自查 + run_command 验证 |
  | 7 | FIX & RETRY | edit_file 修复，最多 3 次，超限说明情况 |
  | 8 | DELIVER | 最终汇报产物清单、文件路径、使用方法 |

  底部加 CRITICAL 强调：「做 PPT」= 生成真实 .pptx 文件，不是聊天框输出文字描述。

**四、 遇到的问题与困难**

* **技术问题：**
  * **SSE 流式输出抖动**：早期 SSE 实现中，流式渲染时 Console 刷新频率不稳定，导致输出在视觉上不平滑。
  * **已尝试的解决方案：** 通过调整 Console 的刷新策略和缓冲区管理，最终实现平滑输出。
  * **需要帮助或待解决：** 无。
* **非技术问题：**
  * Git 操作熟练度不高：在 pull、rebase、stash pop 等操作中遇到冲突和 Vim 编辑器卡住的问题。
  * **已采取的措施：** 通过实践掌握了 `git stash` → `pull` → `checkout -b` → `stash pop` 的工作流，以及 `:wq` 退出 Vim、`git merge --abort` 取消合并等操作。后续建议使用 `--no-edit` 参数避免卡在 Vim 编辑器中。
